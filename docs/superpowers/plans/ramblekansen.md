LIFAIランブルを以下仕様で改修してください。

【目的】
現状は参加時RPの累計で勝敗がほぼ読めてしまい、観戦モードの意味が薄い。
そのため、日次BP報酬を「RPを重みにした抽選方式」に変更し、週次EP報酬は現行維持にする。

【完成仕様】
1. 参加は100BP、1日1回
2. 参加時にRPを内部確定
3. RP計算式：
   RP = 100 + levelBonus + equipBonus + randomFactor
   - levelBonus = level * 2
   - equipBonus = 装備合計（上限50）
   - randomFactor = 0〜50
4. RPは参加直後には詳細公開しない
5. 毎日の締切後、その日の参加者から日次BP報酬当選者を決定
6. 当選方式は重み付き非復元抽選
7. 重みは RPそのままではなく圧縮し、
   weight = sqrt(RP)
   を基準にする
8. 1位〜5位まで順番に抽選し、当選者は都度除外
9. 日次BP報酬は以下固定
   - 1位 1000BP
   - 2位 700BP
   - 3位 400BP
   - 4位 250BP
   - 5位 200BP
10. 週次EP報酬（月〜金累計RPランキング）は現行維持
11. 観戦モードは「勝敗を決めるUI」ではなく、「確定済みの日次抽選結果を演出表示するUI」に変更
12. 観戦中はRP詳細を表示しない
13. 観戦後に必要なら上位者RPや統計を表示可能
14. 抽選結果は再現可能なseed方式で保存する
15. seedは date + secretSalt ベースで生成し、外部から予測不能にする
16. 日次報酬配布処理は冪等にし、同日二重配布不可
17. 台帳(wallet_ledger)に日次BP報酬、週次EP報酬を記録する

【実装要件】
- 既存構造を壊さない
- 削除・省略せず、追加と最小修正で対応
- 既存の週次報酬処理は維持
- 新規 action / route / sheet が必要なら追加
- 実装後、どのファイルをどう変更したか一覧化
- 必要なら前後コードが分かるパッチ形式で出力
