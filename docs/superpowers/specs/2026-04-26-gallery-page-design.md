# /gallery ページ設計仕様

## 概要

ログイン前ユーザー向けに LIFAI の各サービスのデモ・サンプルを展示する静的ギャラリーページ。
Google AdSense 将来対応を見越したセマンティック HTML・テキスト量重視のマガジン型レイアウト。

- **ルート:** `app/gallery/page.tsx`
- **認証:** 不要（公開ページ）
- **レンダリング:** サーバーコンポーネント（SEO最大化）
- **デモ内容:** 静的サンプル（ハードコード）

---

## ページ全体構造

```
/gallery
├── [1] ページヘッダー
├── [2] スティッキーアンカーナビ（GalleryNav.tsx）
├── [3] サービスセクション × 11
│     ├── <!-- ad-slot-1 --> ← 将来の AdSense 枠（現在は空 div）
│     ├── セクション群...
│     └── <!-- ad-slot-N --> × 3〜4箇所
└── [4] フッターCTA
```

## セクション1: ページヘッダー

```
← 最初に戻る（/へのリンク）         [ログイン] [参加申請]

LIFAI でできること
AI×副業の体験を、まず見てみよう
```

## セクション2: スティッキーアンカーナビ

**コンポーネント:** `components/GalleryNav.tsx`（`"use client"`）

- スクロールで上部に固定（`sticky top-0 z-30`）
- PC: 全タブ横並び / スマホ: 横スクロール対応
- `IntersectionObserver` で現在表示中セクションを検知 → アクティブタブをハイライト

## セクション3: サービスセクション一覧

### レイアウトパターン定義

**パターンA（説明左 / サンプル右）**
- 対象: 音楽生成・BGM生成・note記事・Music Boost
- 左カラム: サービス名バッジ・見出し・説明文・できること箇条書き・対象ランク・CTAリンク
- 右カラム: サンプル画像（スクショ or 差し込み画像）
- モバイル: 縦積み（上: テキスト / 下: 画像）

**パターンB（全幅グリッド）**
- 対象: 画像生成・ガチャ
- 上部: 見出し＋説明文
- 下部: サンプル画像 3列グリッド（モバイルは2列）

**パターンC（テキストカード型）**
- 対象: 占い・ミニゲーム・マーケット・ラジオ・LIFAネコ
- 上部: 見出し＋説明文
- 下部: サンプル出力をカード形式で表示（ハードコードされたデモテキスト）

---

| # | id | サービス名 | パターン | サンプル内容 |

## AdSense 対応方針

現時点では広告タグは置かない。以下の空 div を各セクション間に配置し、将来 AdSense タグに差し替えられるよう予約する。

```tsx
{/* ad-slot */}
<div className="w-full" aria-hidden="true" />
```

配置場所（案）: セクション3・5・8・10 の直後

## 既存コードとの整合

- デザイン言語: `rounded-2xl bg-white shadow-sm`（`/start` と統一）
- 背景色: `bg-neutral-50`（`/start` と統一）
- ナビゲーション: トップページ `/` からリンクを追加（任意）
- 画像: `public/` 配下の既存素材を優先使用（`/aiimage.png`, `/lifai/icon/*.png` など）
- 未用意の画像はグレーのプレースホルダー枠で実装し、後から実画像に差し替えできる構造にする
