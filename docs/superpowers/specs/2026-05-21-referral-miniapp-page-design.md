# 紹介ミニアプリ 独立ページ — 設計書

**作成日:** 2026-05-21  
**対象:** LIFAI (Next.js 14 App Router + GAS バックエンド)  
**前提:** 2026-05-08 の設計書（`2026-05-08-referral-dashboard-design.md`）で実装済みの `/api/referral/dashboard` と GAS `my_referral_dashboard` action をそのまま流用する。

---

## 概要

`/top` のミニアプリグリッドから遷移できる独立した紹介ダッシュボードページ `/referral-app` を新設する。  
あわせて `/top` 内の折りたたみ式 `ReferralDashboardCard` を削除し、アプリグリッドのエントリーに置き換える。

---

## ゴール

| ゴール | 詳細 |
|--------|------|
| 紹介コードの一発共有 | QR コード・リンクコピー・コードコピーを最上部に固定表示 |
| サマリーの可視化 | 累計人数・累計報酬・今月報酬・先月報酬を 2×2 グリッドで表示 |
| 月別報酬の把握 | 月ごとのアコーディオンで報酬内訳を確認できる |
| 紹介した人の確認 | プラン・入会日をリスト表示（ID は部分マスク） |
| ツリーの可視化 | 自分を起点にした紹介ツリーを SVG/CSS で描画 |

---

## アーキテクチャ

```
[/top アプリグリッド「リファラ」タップ]
        |
        v
[/referral-app ページ（新規）]
        |
        | ページロード時に 1 回 fetch
        v
[POST /api/referral/dashboard]  ← 既存 API ルートをそのまま使用
        |
        v
[GAS: my_referral_dashboard]    ← 既存 action をそのまま使用
        |
        v
{
  ok: true,
  my_ref_code: "XXXXXX",
  referrals: [{ login_id, plan, approved_at }],
  bonuses: [{ ts, kind, amount, memo }],
  total_bonus: number
}
```

**認証:**
- `localStorage` の `addval_auth_v1` から `id` + `token` を取得
- パスワードは `sessionStorage` の `addval_auth_secret_v1` から取得（`getAuthSecret()`）
- 未ログイン or 認証失敗 → `/login` へリダイレクト

**GAS・API ルートの変更:** なし（既存をそのまま流用）

## ページ構成（`/referral-app`）

カード積み上げ（Approach C）。上から順に以下のセクションが並ぶ。

```
┌────────────────────────────────┐
│  ← 戻る（/top へ）             │  ← ヘッダー
├────────────────────────────────┤
│  ① シェアカード                 │  ← 常時表示
├────────────────────────────────┤
│  ② サマリーカード（2×2）        │
├────────────────────────────────┤
│  ③ 月別報酬（アコーディオン）   │
├────────────────────────────────┤
│  ④ 紹介した人リスト             │
├────────────────────────────────┤
│  ⑤ 紹介ツリー                  │  ← referrals が 1 件以上の場合のみ表示
└────────────────────────────────┘
```

---

## セクション詳細

### ① シェアカード

最上部に固定表示。ページを開いた瞬間にシェアできる状態にする。

**表示内容:**
- 紹介コード（大きめフォント）
- 「コードをコピー」ボタン
- 「リンクをコピー」ボタン（`/purchase?refCode=XXXXXX`）
- QR コード（`qrcode.react` で生成、タップで拡大モーダル）
- 「紹介プログラムとは？」リンク → 既存の `/referral` ページへ

**UX:**
- コピー成功時は「コピーしました ✓」に一時変化（1.5 秒後に戻る）
- データ取得中は QR・コードエリアをスケルトン表示

---

### ② サマリーカード（2×2 グリッド）

| 位置 | 指標 | データソース |
|------|------|-------------|
| 左上 | 紹介した人（累計） | `referrals.length` |
| 右上 | 紹介報酬合計 | `total_bonus` |
| 左下 | 今月の報酬 | `bonuses[]` を当月でフィルタして合計 |
| 右下 | 先月の報酬 | `bonuses[]` を前月でフィルタして合計 |

- 月の計算はフロント側で `ts` を Date 変換してフィルタ（GAS 変更なし）
- 金額表示は `$XX.XX` 形式

---

### ③ 月別報酬（アコーディオン）

- `bonuses[]` を `ts` の年月でグルーピング
- 各月を折りたたみ行として表示（例：「2026年5月  $26.00  ▼」）
- 最新月はデフォルト展開、過去月は折りたたみ
- 展開時のテーブル列：日付 | 種別 | 金額 | メモ
  - `kind` 表示変換: `referral_bonus` → 「紹介報酬」、`referral_entry` → 「入会報酬」
  - メモには `login_id` 末尾 3 桁を含める（例：「...XY1 入会」）
- ボーナスがゼロ件の場合はセクション自体を非表示

---

### ④ 紹介した人リスト

テーブル表示。

| 列 | 内容 | 備考 |
|----|------|------|
| ID | `AB***XY` | 前 2 文字 + `***` + 後 2 文字 |
| プラン | `$67` | |
| 入会日 | `2026/05/10` | `approved_at` を `YYYY/MM/DD` にフォーマット |
| ステータス | `✅ 承認済み` | 現状 `approved` のみ返却 |

- 6 件以上は最初の 5 件を表示 → 「もっと見る」で全件展開
- 0 件の場合は「まだ紹介した人はいません」メッセージ + シェアカードへの「コードをシェアする」ボタン

---

### ⑤ 紹介ツリー

- `referrals` が 1 件以上の場合のみ表示（0 件は非表示）
- SVG または CSS で描画（外部ライブラリ追加なし）
- 構造（現行の通常版 LIFAI は 1 段のみ）:

```
          あなた
      ┌────┼────┐
   AB***  CD***  EF***
```

- 自分のノード：「あなた」と表示
- 紹介した人のノード：`AB***XY`（前 2 文字 + `***` + 後 2 文字）
- 人数が 7 人超の場合は横スクロール対応

## エラーハンドリング

| ケース | 挙動 |
|--------|------|
| 未ログイン | `/login` へリダイレクト |
| API エラー（5xx） | 「データの取得に失敗しました。再試行してください。」＋リトライボタン |
| `ok: false, reason: "pending"` | 「アカウントが承認されていません」メッセージ |
| `bonuses` 空 | サマリーの金額は `$0.00` 表示、月別セクションは非表示 |
| `referrals` 空 | リストは「まだいません」メッセージ、ツリーは非表示 |

## 除外スコープ（今回対象外）

- GAS・API ルートの変更（既存流用のため不要）
- 月の紹介人数上限・残り枠の表示（上限撤廃予定のため不実装）
- 紹介コードの変更機能
- プッシュ通知・メール通知
- 多段ツリー（/5000 版は別仕様）
