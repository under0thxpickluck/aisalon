# やること

## refcodeから紹介者を自動振り分け（admin/finance）

### 背景
- `applies`シートの`ref_code`列に紹介コードが入っているユーザーがいる
- しかし紹介者の自動解決（`referrer_login_id`への反映）は「承認時」にしか走らない
- 過去ユーザーや後からref_codeを手入力した場合は反映されないまま
- `/admin/finance` の紹介ツリーで手動ドラッグ以外の方法で一括更新したい

### 実装内容

#### 1. GAS: `ref_backfill_from_refcode` アクション（`gas/Code.gs`）

- `applies`シート全行を走査
- 条件: `ref_code`が入っていて、かつ`referrer_login_id`が空の行のみ対象（既存設定は上書きしない）
- 各行の`ref_code`で`my_ref_code`を検索 → 一致したら紹介者を解決
- 更新カラム: `referrer_login_id` / `referrer_2_login_id` / `referrer_3_login_id` / `ref_path`
- `ref_events`シートにも監査ログを記録（`appendRefEvent_`を使用）
- レスポンス: `{ ok: true, updated: N, skipped: N, unmatched: N }`
  - `updated`: 紹介者を新たに設定した件数
  - `skipped`: `referrer_login_id`が既に設定済みでスキップした件数
  - `unmatched`: `ref_code`はあるが対応する`my_ref_code`が見つからなかった件数

#### 2. Next.js APIルート: `app/api/admin/ref-backfill/route.ts`

- `POST /api/admin/ref-backfill`
- Admin Basic Auth（`middleware.ts`が自動でカバー）
- GAS `ref_backfill_from_refcode` アクションを `GAS_ADMIN_KEY` 付きで呼び出す
- レスポンスをそのままフロントへ返す

#### 3. UI: `app/admin/finance/TreeTab.tsx` にボタン追加

- ボタンラベル: 「🔗 refcodeから紹介者を更新」
- 押下 → `POST /api/admin/ref-backfill` を呼び出す
- ローディング中はボタンを無効化
- 結果を画面に表示:
  - 成功: 「✅ XX件更新、YYスキップ（設定済み）、ZZ件不一致（ref_code未登録）」
  - エラー: エラーメッセージを赤字で表示
- ボタン位置: TreeTabの「ツリー再生成」ボタン群の近く

### 注意点
- `referrer_login_id`が既に設定されている行は**上書きしない**（手動設定・ドラッグ割当を保護）
- 強制上書きが必要な場合は別途 `force=true` パラメータの追加を検討
- `ref_tree_build` は別アクション — backfill後に手動でツリー再生成が必要
