# /5000 メールフロー設計書

## 現状の構造

### ユーザー導線

```
/5000（LP）
  └→ /5000/apply（申請フォーム）
       └→ /5000/confirm（確認画面）
            ├→ GAS: apply_5000（申請行を作成）
            └→ NOWPayments invoice 発行 → 外部決済画面
                 └→ /5000/purchase-status（ポーリング画面）
                      └→ GAS: payment_update_5000（IPN受信で更新）
                           └→ 管理者が /5000/admin で手動承認
                                └→ GAS: admin_approve_5000
                                     └→ 【現在は1通だけ】パスワードリセットメール送信
```

### 現在送られるメール（1通のみ）

| タイミング | 送信元 | 内容 |
|---|---|---|
| 管理者が `admin_approve_5000` を実行したとき | GAS | パスワードリセットトークン付きのログイン案内メール（1通） |

メールが届いたことは `/5000/purchase-status` 画面の `mail_sent` フラグで確認できる。  
届かなかった場合は同画面の「認証メールを再送する」ボタンから `reset_resend_5000` を呼べる。

---

## 変更後の構造（2段メール）

### 全体フロー（変更後）

```
NOWPayments 決済完了（payment_update_5000 で isPaid = true）
  └→ 【Email 1】月額サブスク案内メール（自動送信）

管理者が /5000/admin で手動承認（admin_approve_5000）
  └→ 【Email 2】ログイン情報メール（パスワード + 暗証番号）
```

---

### Email 1 ── 月額サブスク案内メール

**送信タイミング**: GAS の `payment_update_5000` アクション内で `isPaid = true` になった瞬間（自動）

**送信先**: 申請フォームに入力したメールアドレス

**目的**: 暗号資産の一括払いが完了したことを伝え、月額クレジットカード契約の手続きへ誘導する

**メール内容（案）**:

```
件名: 【LIFAI】お支払い確認 ─ 次のステップ：月額サポート契約のお手続きへ

{name} 様

この度は LIFAI Limited Access へのご参加ありがとうございます。
${plan} プランのお支払いを確認いたしました。

■ 次のステップ：月額サポート契約（¥9,800 / 月）

サービスのご利用には、月額クレジットカード契約の手続きが必要です。
下記リンクよりお手続きをお願いします。

  [月額契約お申し込みリンク]

※ 月額契約が完了したことを確認後、管理者がアカウントを承認いたします。
※ 承認後に別途ログイン情報をお送りします。

ご不明な点はサポートまでご連絡ください。
LIFAI サポートチーム
```

**実装箇所（GAS）**: `payment_update_5000` 関数内、`isPaid = true` の分岐に追記

---

### Email 2 ── ログイン情報メール

**送信タイミング**: 管理者が `admin_approve_5000` を実行したとき（手動承認）

**送信先**: 申請フォームに入力したメールアドレス

**目的**: ログインIDとパスワードリセットリンク、および暗証番号を伝える

**メール内容（案）**:

```
件名: 【LIFAI】アカウント承認完了 ─ ログイン情報

{name} 様

アカウントの審査が完了しました。以下の情報でログインしてください。

■ ログインID: {login_id}
■ 初回パスワード設定リンク（72時間有効 / 1回限り）:
  https://lifai.vercel.app/reset?token={reset_token}

■ 暗証番号（4桁）: {pin}
  ※ 暗証番号は{用途TBD}の際に必要です。メモして安全な場所に保管してください。

ログイン後は下記 URL からダッシュボードへアクセスできます:
  https://lifai.vercel.app/top

LIFAI サポートチーム
```

**実装箇所（GAS）**: `admin_approve_5000` 関数内、現在の `sendResetMail_` を置き換え or 拡張

## 実装に向けた確認事項（未決定）

| 項目 | 状況 | 確認が必要な内容 |
|---|---|---|
| 月額サブスクの申し込みリンク | 未定 | Stripe / 外部サービスの URL をどこに置くか |
| 暗証番号（PIN）の定義 | 未定 | 何桁か、何のために使うか、GAS で生成するかユーザー設定か |
| Email 1 の送信条件 | 要確認 | `isPaid = true` が複数回飛んでくる可能性があるため重複送信防止が必要 |
| Email 2 の差分 | 要確認 | 現行の `sendResetMail_` を流用するか、完全に作り直すか |
| GAS のメール送信 | 既存利用中 | `GmailApp.sendEmail` or `MailApp.sendEmail` を使用中 |

## GAS アクション対応表（変更後）

| action | 呼び元 | メール |
|---|---|---|
| `apply_5000` | `/api/5000/apply` | 送信なし |
| `payment_update_5000` | `/api/5000/nowpayments/ipn` | **Email 1** を送信（isPaid=true 初回のみ） |
| `admin_approve_5000` | `/api/5000/admin/approve` | **Email 2** を送信（パスワード + PIN） |
| `reset_resend_5000` | `/api/5000/reset/resend` | Email 2 を再送（既存機能） |
