## 1. ビジネス概要

### コンセプト

**LIFAI（ライファイ）** は「AI × LIFE × 副業」をコンセプトとした、日本向けの **AI実践型オンラインサロン** です。

```
AI の力を借りて、
毎日の仕事・副業・学習を加速させるコミュニティ
```

### ターゲット

| セグメント | 特徴 |
|---|---|
| AI初学者 | AIツールを使い始めたい会社員・フリーランス |
| 副業志望者 | AI活用で収益化を目指す人 |
| クリエイター | AI生成コンテンツを活用したい人 |
| 起業家予備軍 | AIを使ったビジネス構築に興味がある人 |

### ビジネスモデル

```
収益源:
1. メンバーシップ入会費（USDT一括払い）
2. マーケットプレイス手数料（5.5%）
3. 将来: 継続課金・コンテンツ販売
```

### コアバリュー

1. **実践重視** — 使えるテンプレート・ツールを即日提供
2. **コミュニティ** — メンバー同士でコンテンツを売買・共有
3. **AI民主化** — 難しい AI を誰でも使えるように抽象化
4. **紹介ネットワーク** — 紹介者にも恩恵が返る報酬設計

## 4. BP / EP ポイントシステム

### BP（Bonus Point）

**性格:** コミュニティ内通貨（消費型）

#### 取得方法

| 取得方法 | タイミング | 量 |
|---|---|---|
| プラン購入 | 承認時に一括付与 | プランに応じた量 |
| 紹介配当 | 紹介成功時 | 変動（紹介ツリーによる） |
| 未受取 BP 受け取り | `/top` で通知→受取 | 変動 |

#### 使用方法

| 使用方法 | 消費量 |
|---|---|
| 音楽生成（旧） | プランに応じた量 |
| 曲作成（新） | 10 BP / 曲 |
| マーケット購入 | 商品価格分 |

#### BP 残高確認

- `/top` ダッシュボードの `WalletBadge` コンポーネント
- API: `GET /api/wallet/balance`

---

### EP（Energy Point）

**性格:** 活動実績型ポイント（別用途）

#### 取得方法
- 紹介ツリーや活動実績に応じて付与（詳細は GAS 側で管理）

#### 使用方法
- マーケットプレイスでの購入（BP と並行する通貨として機能）

---

### 未受取 BP 受け取り機能

```
/top ページ表示時:
  → GET /api/user/pending-bp
  → 未受取 BP あり: BPGrantModal を表示
  → ユーザーが「受け取る」:
    → POST /api/user/claim-bp
    → BP がウォレットに加算
```

## 5. 紹介（リファラル）プログラム

### 概要

各メンバーに固有の紹介コードが発行され、紹介経由で入会した新規メンバーが発生した場合に、紹介者にボーナスが付与される。

### 紹介コード形式

```
R-lifai_{applyId の数字部分}
例: R-lifai_1709876543210
```

### 紹介コードの場所

- `/top` ダッシュボード → `ReferralCard` コンポーネント
- API: `POST /api/me` → `my_ref_code` フィールド

### 紹介ツリー構造

```
最大 3 段階の追跡:
  Aさん（紹介者）
  └─ Bさん（Aの紹介で入会）
     └─ Cさん（Bの紹介で入会）
        └─ Dさん（Cの紹介で入会）

各ユーザーの applies シート:
  referrer_login_id: 直接紹介者
  ref_path: 紹介チェーン全体
```

### 紹介配当の仕組み

- GAS 側の `planToGrant_()` 関数で配当量を管理
- `wallet_ledger` シートに取引ログを記録
- `ref_events` シートに紐づけ監査ログを記録

### 紹介ツリー再生成

管理者が `ref_tree_build` アクションを実行すると:
1. `ref_tree` シートを全消去
2. `applies` シートから全ユーザー情報を読み込み
3. 紹介ツリーを再計算して `ref_tree` シートに書き込み

## 6. ユーザー登録・オンボーディングフロー

### ステップ詳細

#### Step 0: ホームページ（/）

- プレセール情報（カウントダウン・調達バー）を確認
- 「権利購入」ボタンから申請開始

#### Step 1: プラン選択（/purchase）

1. 5段階プランから1つ選択
2. `applyId` 自動生成（`lifai_{Date.now()}`）
3. GAS に仮申請行を作成（`apply_create`）
4. NOWPayments のインボイスを作成
5. 外部決済ポータルへリダイレクト
6. USDT（TRC20）で支払い

#### Step 2: 申請フォーム（/apply）

入力が必要な情報:

| 項目 | 必須 | 説明 |
|---|---|---|
| メールアドレス | ✅ | ログイン情報受信先 |
| 氏名 | ✅ | 本名 |
| 氏名（カナ） | ✅ | フリガナ |
| Discord ID | ✅ | コミュニティ参加用 |
| 年代 | ✅ | 10代〜60代以上 |
| 都道府県 | ✅ | 全47都道府県から選択 |
| 市区町村 | ✅ | テキスト入力 |
| 職業 | ✅ | 選択肢から選ぶ |
| 紹介者名 | ❌ | 任意 |
| 紹介者ID | ❌ | 紹介コード入力 |

#### Step 3: 確認・送信（/confirm）

1. 入力内容の最終確認
2. 支払い完了を確認（`GET /api/apply/status`）
3. 申請を GAS に送信（`POST /api/apply`）
4. `/pending` へリダイレクト

#### 審査・承認

**自動承認（条件あり）:**
- NOWPayments IPN 受信時
- 実際の支払い額 ≥ 期待支払い額 × 98%
- 自動でログイン情報生成・メール送信

**手動承認（管理者）:**
- 管理者が `/admin` で申請一覧を確認
- 「承認」ボタンで個別に承認
- GAS が自動でログイン情報生成・メール送信

#### 初回ログイン

1. 承認メールにパスワードリセットリンクが届く
2. `/reset?token=...` でパスワードを設定（72時間・1回限り）
3. `/login` でログイン
4. `/top` ダッシュボードへ

## 9. 音楽・コンテンツ生成サービス

### 音楽生成（旧: /music）

シンプルなテーマ入力→BGM/ループ生成。全プランで利用可能。

### 音楽・曲生成（新: /music2）

より高品質な楽曲を3ステップで生成するサービス。

**消費コスト:** 10 BP / 曲

**生成フロー:**

```
ステップ 1: テーマ入力
  - テーマ（テキスト自由入力）
  - ジャンル（10種: ポップ/ロック/ジャズ等）
  - 雰囲気（8種: さわやか/クール/エモい等, 複数選択可）

ステップ 2: 歌詞確認・編集
  - AIが歌詞を自動生成（タイトル + Verse/Chorus/Bridge）
  - ユーザーが自由に編集可能

ステップ 3: 楽曲構成確認
  - BPM / キー / セクション構成 / サビのポイント

ステップ 4: 音楽生成（2〜5分）
  - 段階別進捗表示:
    イントロ → Verse → サビ → アウトロ → 仕上げ
  - 完成後: 再生 + WAV ダウンロード
```

**タイムアウト:** 最大10分（600秒）

## 10. 決済・返金ポリシー

### 決済方法

| 方法 | 詳細 |
|---|---|
| 通貨 | USDT (TRC20) |
| プラットフォーム | NOWPayments |
| 許容誤差 | 期待額の -2% まで自動承認 |

### 支払いフロー

1. NOWPayments インボイスへリダイレクト
2. USDT（TRC20）で支払い
3. IPN Webhook 受信後、GAS で自動承認 or 管理者が手動承認

### 返金ポリシー

```
役務の提供が開始される前であれば、全額 USDT での返金を受け付けます。

ただし、以下の手数料はご負担いただきます:
- ブロックチェーンネットワーク手数料
- NOWPayments 処理手数料

役務提供開始後（承認・ログイン情報発行後）は原則返金不可。
```

### 決済エラー時の対応

- 支払い額が期待額の 98% 未満: 自動承認されず、管理者確認待ち
- 支払い通貨が異なる場合: 管理者判断
- 二重支払い: 管理者に問い合わせ（LINE）

## 11. 管理者運営フロー

### 日常業務

#### 申請確認・承認

1. `/admin`（Basic Auth必要）にアクセス
2. 申請一覧で `paid` ステータスの申請を確認
3. 支払い内容（金額・通貨）を確認
4. 問題なければ「承認」ボタンをクリック
5. GAS がログイン情報生成 + パスワードリセットメールを自動送信

#### BP付与（売却申請対応）

1. 売却申請一覧（`/api/admin/sell-requests`）を確認
2. 申請内容を確認
3. `POST /api/admin/grant-bp` で BP を付与

#### 紹介ツリー更新

必要時に GAS から `ref_tree_build` を実行:
- `ref_tree` シートを全消去
- 全ユーザーの紹介ツリーを再構築

### 管理者認証情報

| 項目 | 設定場所 |
|---|---|
| ユーザー名 | `ADMIN_USER` 環境変数 |
| パスワード | `ADMIN_PASS` 環境変数 |
| 対象パス | `/admin/*`, `/api/admin/*` |
